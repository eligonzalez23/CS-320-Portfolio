## Reflection

### How can I ensure that my code, program, or software is functional and secure?

I can make sure my code is functional by testing it often and checking that it meets the requirements before I call it finished. In this course, I learned that unit tests are important because they help catch problems early instead of waiting until the whole program is done. I can also make my software more secure by validating input, not allowing invalid data, and thinking about how a user might accidentally or intentionally enter something wrong.

### How do I interpret user needs and incorporate them into a program?

I interpret user needs by carefully reading the requirements and turning them into specific features in the program. For example, if the requirement says a contact ID cannot be longer than 10 characters, then my code needs to enforce that rule. I also learned that tests should connect back to the requirements, because that helps prove the program is doing what the user asked for.

### How do I approach designing software?

I approach software design by starting with the requirements first, then breaking the program into smaller parts that are easier to build and test. I try to keep each class focused on one main purpose so the code is easier to understand. I also think testing should be part of the design process, not something added only at the end.
